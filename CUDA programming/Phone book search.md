

### প্রোগ্রামটা কী করে?
এটা একটা **CUDA (GPU) প্রোগ্রাম** যেটা ফোনবুক ফাইল (`phonebook1.txt`) থেকে নাম খুঁজে বের করে।  
তুমি যে নামের অংশ দিবে (যেমন "TANBIR"), সেটা দিয়ে নামের মধ্যে সাবস্ট্রিং (substring) মিল আছে কি না চেক করে।  
GPU-তে সমান্তরালভাবে (parallel) অনেক কন্টাক্ট একসাথে চেক করে দ্রুত রেজাল্ট দেয়।  
শেষে মিলে যাওয়া কন্টাক্টগুলো নাম অনুযায়ী সাজিয়ে প্রিন্ট করে।

### ধাপে ধাপে পুরো প্রক্রিয়া (বাংলায়)

1. **হেডার ও ম্যাক্রো**
   - `#include <bits/stdc++.h>` → সব স্ট্যান্ডার্ড লাইব্রেরি একসাথে নেয়
   - `#include <cuda.h>` ও `<cuda_runtime.h>` → CUDA-র জন্য
   - `#define MAX_STR_LEN 50` → নামের সর্বোচ্চ দৈর্ঘ্য ৫০ অক্ষর

2. **স্ট্রাকচার তৈরি**
   - `Contact` → নাম + নম্বর রাখে
   - `ResultContact` → শুধু মিলে যাওয়া কন্টাক্ট রাখে + নাম দিয়ে সাজানোর জন্য `<` অপারেটর

3. **GPU-তে চলা ফাংশন (`__device__ check`)**
   - এটা GPU-তে চলে
   - একটা নামের ভিতরে সার্চ স্ট্রিং আছে কি না চেক করে
   - উদাহরণ: "TANBIR" খুঁজলে "MD TANBIR RAHMAN" এ মিলবে

4. **CUDA কার্নেল (`searchPhonebook`)**
   - `__global__` → হোস্ট (CPU) থেকে কল করা যায়
   - প্রতিটি থ্রেড একটা কন্টাক্ট চেক করে
   - `blockIdx.x`, `threadIdx.x` দিয়ে থ্রেডের ID বের করে
   - মিললে `d_results[idx] = 1` করে

5. **main ফাংশনে কী কী হয়**
   - কমান্ড লাইন থেকে নেয়: সার্চ স্ট্রিং + থ্রেড সংখ্যা  
     উদাহরণ: `./search_phonebook TANBIR 256`
   - ফাইল পড়ে (`phonebook1.txt`)
     - লাইন ফরম্যাট: `"নাম","নম্বর"`
     - কমা দিয়ে আলাদা করে নাম ও নম্বর নেয়
   - সব নাম একটা ফ্ল্যাট অ্যারেতে রাখে (`h_names`)
   - GPU-তে মেমরি বরাদ্দ করে (`cudaMalloc`)
   - ডেটা কপি করে GPU-তে (`cudaMemcpy`)
   - কার্নেল চালায় → `<<<blocks, threads>>>`
   - ফলাফল ফিরিয়ে আনে
   - যেগুলো মিলেছে সেগুলো সংগ্রহ করে
   - নাম অনুযায়ী সাজায় (`std::sort`)
   - প্রিন্ট করে
   - মেমরি ফ্রি করে (`free`, `cudaFree`)

### ব্যবহৃত গুরুত্বপূর্ণ ফাংশনগুলোর ব্যাখ্যা (সংক্ষেপে)

| ফাংশন/API                  | কাজ                                      | প্যারামিটার (সংক্ষেপে)                          |
|----------------------------|------------------------------------------|-------------------------------------------------|
| `std::getline`             | ফাইল থেকে লাইন পড়া                      | ফাইল, স্ট্রিং                                   |
| `string::find`             | কমা খুঁজে বের করা                        | খুঁজতে হবে স্ট্রিং                              |
| `string::substr`           | অংশ কেটে নেয়া                           | শুরুর পজিশন, দৈর্ঘ্য                            |
| `strncpy`                  | নাম কপি (নিরাপদে)                        | ডেস্টিনেশন, সোর্স, সাইজ                        |
| `cudaMalloc`               | GPU-তে মেমরি বরাদ্দ                      | পয়েন্টার, সাইজ (বাইট)                          |
| `cudaMemcpy`               | CPU ↔ GPU ডেটা কপি                       | ডেস্ট, সোর্স, সাইজ, দিক (HostToDevice / DeviceToHost) |
| `<<<blocks, threads>>>`    | কার্নেল চালানো                            | গ্রিড সাইজ, ব্লক সাইজ                          |
| `cudaDeviceSynchronize`    | GPU শেষ হওয়া পর্যন্ত অপেক্ষা            | কোনো প্যারামিটার নেই                            |
| `std::sort`                | লিস্ট সাজানো                             | শুরু ও শেষ ইটারেটর                              |
| `cudaGetLastError`         | CUDA-র শেষ এরর চেক করা                   | কোনো প্যারামিটার নেই                            |

### কম্পাইল ও রান করার কমান্ড (যেটা কমেন্টে ছিল)
```bash
nvcc -arch=compute_75 -code=sm_75 search_phonebook_single_list.cu -o search_phonebook
time ./search_phonebook TANBIR 256
```

এই প্রোগ্রামটা বড় ফোনবুকের জন্য খুব দ্রুত কাজ করে কারণ GPU-তে হাজার হাজার থ্রেড একসাথে চেক করে।
